Por qué y Para qué

Contexto Estratégico

El Problema & La Oportunidad

La evaluación manual de las solicitudes de cambio (CRs) en ServiceNow genera cuellos de botella operativos y variabilidad en la aplicación de las políticas corporativas del banco. El proyecto **ITSO Companion / Change Intelligence Assistant** soluciona esto a través de automatización inteligente, analizando el riesgo de manera consistente e instantánea.

Objetivo de la POC

Validar la viabilidad de la integración segura entre ServiceNow y los modelos cognitivos de la AI Platform usando un servidor MCP (Model Context Protocol), operando de forma controlada sobre ambientes aislados en GCP antes de proceder a la industrialización multi-asistente.

Plataformas y Componentes Participantes

AI Platform

Ecosistema corporativo de inteligencia artificial. Alojamiento de modelos y orquestación de prompts autorizados.

Agent Hub

Orquestador centralizado de agentes autónomos y asistentes inteligentes aprobados para uso en el banco.

MCP Server

Model Context Protocol Server. Habilita una interfaz estándar para que el LLM consuma datos de ServiceNow de forma segura.

ServiceNow

Sistema de registro origen de las Change Requests (CRs) evaluadas contra las políticas de TI.

Google Cloud Run

Hospeda la interfaz ejecutiva y el backend del microservicio mediante arquitectura serverless, ágil y escalable.

Firestore & Cloud Storage

Persistencia NoSQL ligera para metadatos/auditoría y buckets seguros de políticas y documentos de referencia.

DSP / Active Directory

Infraestructura de identidad y accesos para la autenticación única corporativa del personal evaluador.
